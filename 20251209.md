## 情報の表現
* アナログ方式
  * 連続的な値で表現(シームレス)
* デジタル方式
  * 離散的な値で表現(0, 1)

## コンピュータのデータ
* あらゆる情報をデジタルデータ(0, 1)で扱う
* トランジスタの閾値電圧を境にアナログ的に(0, 1)判定をしている
  * 高電圧⇒オン、定電圧⇒オフ 

## 情報の単位
* 0, 1 一桁分のデータ⇒1ビット
* 8ビット⇒1バイト

## コンピュータ
* トランジスタの組み合わせでAND/OR/NOTなど論理ゲートを構成
* 論理ゲートの組み合わせで以下を構成
  * 加算器
  * 比較器
  * 分岐回路
  * メモリ回路 
* 命令によりどの回路を使うかを切り替える


## String Pool
* 整数で同じシステム
  * Integer Cache -128～127
  * 同様にBoolean(true, false),Byte,Short,LongにもCacheはある


## コンピュータの五大構成
* 入力装置
* 記憶装置
* 演算装置
* 出力装置
* 制御装置
* コンピュータは入力、記憶、演算、出力という流れを有する
* 同様にプログラムもこの流れを持っている
  * 入力(外部入力または引数)を受け
  * 変数に記憶し
  * 式やメソッドで演算し
  * 戻り値として出力またはコンピュータから出力する

## 文(Statement)と式(Expression)
* 文⇒命令文
* 式は評価し値を返す

## データ型(Javaは静的型付け言語 Static Typed Language)
* Primitive Type
  * byte, short, lint, long, float, double, char, boolean
  * 値そのもの
  * メモリ配置⇒スタック
* Reference Type(オブジェクトへの参照値)
  * Class Type(Enum Type), Interface Type, Array Type
  * オブジェクト(その参照値)
  * メモリ配置⇒ヒープ

## 変数
* 変数宣言時に型の指定が必要
* Primityve Type は値自体 Primitive Value が格納される
* Reference Type は参照値 Reference Value が入る
  * 参照型変数 Reference Variable

## メソッドの引数
* 式が評価された結果として渡される Primitive Value か Reference Value を渡すことができる

## オブジェクト指向とは
* オブジェクトとは
  * 各オブジェクトはそれぞれ何らかの役割・責務を持っている
  * その役割・責務を果たすための情報や機能の集合がオブジェクト
  * データ(状態)とそのデータを操作する処理(振る舞い)をひとまとめにしたもの
    * オブジェクトが保持する情報⇒属性⇒フィールド・プロパティ
    * オブジェクトが持つ機能⇒操作・振る舞い⇒メソッド
  * クラスが設計図でそれをインスタンス化したものがオブジェクト
    * クラスはインスタンスのひな型
    * staticがついているクラスはそのまま利用可能
      * 実行時にstatic領域に展開済みのため
    * staticがないクラスはインスタンス化して利用する

## オブジェクト指向
* カプセル化
* データ隠蔽
  * カプセル化を維持する
* 抽象化・ポリモーフィズム
* 情報隠蔽


## 副作用(side effect)とは
* メソッドや式の実行により外部の状態が変化すること
* 戻り値以外の形で状態を変えること
* 予測しづらいバグの原因となる
  * メソッドを呼ぶだけで知らないうちに外部状態が変わると読みづらい
  * 状態が変わるため同じ入力でも結果が変わる可能性がありテストが困難

## 関数型
* 副作用がない⇒外部状態を変化しない
* 同じ引数なら常に同じ結果を返す
 

## クラス
* メンバ
  * フィールド(変数)
    * クラスフィールド
    * インスタンスフィールド
  * メソッド
    * クラスメソッド(staticあり)
      * 静的メソッド内では静的メンバ(自クラス)のみ利用可能
        * 理由:インスタンス化されているとは限らないから
        * ただし自メソッド内で明示的にインスタンス化すれば利用できる
    * インスタンスメソッド
